# Plan: Custom Employee Ordering in Team Roster

## Overview
Add custom ordering for employees in the team roster. Admins can reorder via drag-and-drop and inline order numbers in the Admin Users page. The roster view respects this custom order. All users have an order number; new users get the last order number + 1.

## Changes

### 1. Database Migration (NEW)
**File:** `netlify/database/migrations/20260704_add_user_order.sql`
- Add `display_order INTEGER NOT NULL DEFAULT 0` column to `users` table
- Add index on `display_order`
- Seed existing users with sequential order based on current alphabetical sort:
  ```sql
  WITH numbered AS (
    SELECT id, ROW_NUMBER() OVER (ORDER BY name) AS rn FROM users
  )
  UPDATE users u SET display_order = n.rn FROM numbered n WHERE u.id = n.id;
  ```

### 2. Backend: `netlify/lib/users.ts` (MODIFY)
- **`getUsers()`**: Use `ORDER BY display_order ASC, name ASC` (no parameter)
- **`getFullUsers()`**: Call `getUsers()` (already returns custom-ordered results)
- **New `reorderUsers(orderedUserIds)`**: Bulk update all display_order values in a transaction
- **New `updateDisplayOrder(id, order)`**: Update single user's order
- **`inviteUser()`**: Set new employee's `display_order` to `MAX(display_order) + 1`
- **`syncIdentityUser()`**: Same — new employees appended at end
- **`updateUser()`**: Accept `displayOrder` field

### 3. Backend: `netlify/functions/users.ts` (MODIFY)
- Add `PATCH` handler: `{ reorder: [id1, id2, ...] }` or `{ id, displayOrder }`
- Extend `PUT` handler to accept `displayOrder`

## Data Flow
```
Admin drags/reorders → UserOrderList updates state → Save → PATCH /users
  → reorderUsers() bulk UPDATE → PostgreSQL
Roster loads → getUsers() → ORDER BY display_order, name → correct order
```
