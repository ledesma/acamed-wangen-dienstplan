# Plan: Extend Day Comments to Support Employee-Specific Comments

## Overview

Extend day comments to support both **global** (per-day) and **employee-specific** comments. Auto-detect type: clicking day header = global, clicking a roster cell = employee-specific. Works even without a shift. Both types display identically: footnote marker on cell(s), comment text in footnote section.

## Data Model Change

### Migration: `netlify/database/migrations/20260703_extend_day_comments.sql` (new file)

Add `user_id` column to `day_comments`:
```sql
ALTER TABLE day_comments ADD COLUMN user_id TEXT REFERENCES users(id) ON DELETE SET NULL;

-- Partial unique indexes: one global + many employee-specific per date
CREATE UNIQUE INDEX idx_day_comments_global ON day_comments(date) WHERE user_id IS NULL;
CREATE UNIQUE INDEX idx_day_comments_employee ON day_comments(date, user_id) WHERE user_id IS NOT NULL;
```

## UI Flow

1. **Admin clicks day header** → global comment editor
2. **Admin clicks roster cell** → employee-specific comment editor (any cell, with or without shift)
3. **Footnote section** shows all comments: global first, then employee comments with "Name: " prefix
4. **Footnote markers** appear on cells: day header for global, employee's cell for employee-specific
5. **Employee personal roster** sees both global footnotes and their own employee comments (with markers on their cells)
